## 5-Minute Inactivity Session Timeout (Dual-Layer)
To ensure maximum security on this identity dashboard:
1. **Client-Side Inactivity Monitor**:
   - Create an `InactivityProvider` component that wraps the dashboard layout.
   - Listen to `mousemove`, `click`, `keydown`, `scroll`, and `touchstart` events.
   - Set a 5-minute (300,000 ms) timer.
   - When the timer fires, call the Better-Auth sign-out endpoint `/api/auth/sign-out` and redirect the browser to `/login?reason=timeout`.
2. **Server-Side Cookie Guard**:
   - On every request or Server Action, the application reads a secure cookie `ei_last_activity` containing the last active timestamp.
   - If `current_time - last_active > 5 minutes`, the session is considered expired. We clear the authentication cookies, delete the session from the DB, and redirect to `/login`.
   - The cookie `ei_last_activity` is updated on the client side periodically (e.g., throttled every 30 seconds) on user interaction.

## Contract Subscriptions & SKU Assignment
- **Lifecycle**: When assigning a `Suscripcion` to an `Empresa`, we select the SKU.
- **Validity start**: The subscription's `fechaInicio` is set to `now`. The `fechaVencimiento` is computed automatically from `fechaInicio` using the SKU's validity `periodo` (e.g., adding 1 month for mensual, 12 months for anual).
- **Immutability**: The subscription edit button will be hidden/removed. A subscription can only be deleted.
- **Soft Delete / History**: When deleted, instead of removing the record from the database, we update its `estado` to `"eliminada"`. The active subscriptions list filters by `estado != "eliminada"`, while the "Historial" tab shows all subscriptions (including those with status `"eliminada"`).

## MFA (Multi-Factor Authentication) Integration (Native TOTP)
To harden access control when users only have email/password credentials:
1. **Plugin Integration**: We will enable the Better-Auth native **two-factor** plugin with TOTP support.
2. **User Flow**:
   - When a user logs in with email and password:
     - Check if they have TOTP enabled.
     - If yes, redirect them to a second-factor step `/login/mfa` to enter the 6-digit TOTP code generated by their authenticator app (Google Authenticator, Microsoft Authenticator, etc.).
     - If no, allow login but display a prominent recommendation banner in their dashboard profile to set up Authenticator App or register a Passkey.
3. **Super Admin management**: Super admins will have the ability to reset a user's MFA/TOTP state if they lose access to their authenticator.

## Telegram Security Alerts & Styled "Sin Acceso" Page Design

### 1. Telegram Notifier Utility (`src/lib/telegram.ts`)
- Implement `sendTelegramAlert(message: string)` that resolves the admin phone number from the `User` table (where `role` is `admin` or `superadmin` and `telefono` is not empty) with a fallback to `+59175213449`.
- Perform a POST request to `http://localhost:3200/api/v1/notifications/send` with headers:
  - `Content-Type: application/json`
  - `x-api-key: super_secret_internal_key_2026`
- Request body: `{ phoneNumber: string, message: string }`.

### 2. Login Failure Hook / Request Interceptor (`src/app/api/[...all]/route.ts`)
- Wrap the Next.js API route handler to check if the route is a POST request to a sign-in path (like `/api/auth/sign-in/email`).
- If the response status code is not 200 (e.g. 400 or 401), parse the request (clone it to avoid disrupting Better Auth) to extract the email, and send a Telegram alert notification detailing:
  - Date and time
  - Email/username used (never log the password!)
  - IP address and User Agent
  - Status code/reason of failure

### 3. Styled "Sin Acceso" Page (`src/app/sin-acceso/page.tsx`)
- Implement a styled Next.js page that fits the dashboard's zinc color system and typography.
- Display the clean message: *"Sin acceso: No tienes permisos para acceder al panel de administración."*
- Ensure it does not expose technical details or debug screens.
- When accessed, query the current session (if any) and trigger a Telegram alert if a user with insufficient permissions attempted to access the admin panel.
